Panduan Deployment Laravel ke Ubuntu Server
===========================================

Dokumen ini merangkum isi deployment.txt lama menjadi alur yang lebih rapi,
detail, dan aman untuk deployment Laravel di Ubuntu Server dengan Nginx,
MySQL, PHP-FPM, dan source code dari GitHub.

Prinsip dasar
-------------
1. Source code utama disimpan di GitHub.
2. Server production hanya dipakai untuk menjalankan aplikasi.
3. File yang boleh diedit langsung di server pada umumnya hanya `.env`.
4. Update aplikasi dilakukan dengan `git pull`, install dependency, build asset,
   migrasi database, lalu refresh cache Laravel.

Contoh asumsi
-------------
- OS server: Ubuntu
- Web server: Nginx
- Database: MySQL
- Folder project: `/var/www/laravel-app`
- User server: `htjogit`
- Group web server: `www-data`
- Branch deploy: `main`
- Domain atau IP server: `192.168.1.8`

Sebelum mulai, sesuaikan placeholder berikut:
- `USERNAME`
- `REPO`
- `TOKEN_GITHUB`
- `laravel-app`
- `laravel_db`
- `laravel_user`
- `password_kuat`
- `192.168.1.8`

1. Update package server
------------------------
Jalankan:

sudo apt update
sudo apt upgrade -y

Tujuan:
- memastikan index package terbaru
- memasang update keamanan dan bugfix

2. Install service utama
------------------------
Install Nginx dan MySQL lebih dulu:

sudo apt install nginx mysql-server -y

Aktifkan dan jalankan servicenya:

sudo systemctl enable nginx
sudo systemctl start nginx
sudo systemctl enable mysql
sudo systemctl start mysql

Cek status:

systemctl status nginx
systemctl status mysql

3. Secure MySQL
---------------
Jalankan hardening dasar:

sudo mysql_secure_installation

Rekomendasi jawaban:
- Remove anonymous users: Y
- Disallow root login remotely: Y
- Remove test database: Y
- Reload privilege tables now: Y

4. Buat database dan user MySQL
-------------------------------
Masuk ke MySQL:

sudo mysql

Lalu jalankan:

CREATE DATABASE laravel_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'laravel_user'@'localhost' IDENTIFIED BY 'password_kuat';
GRANT ALL PRIVILEGES ON laravel_db.* TO 'laravel_user'@'localhost';
FLUSH PRIVILEGES;
EXIT;

Fungsi langkah ini:
- membuat database aplikasi
- memisahkan akses database per aplikasi
- menghindari penggunaan user root untuk Laravel

5. Install PHP dan dependency Laravel
-------------------------------------
Pasang PHP-FPM dan extension yang umum dipakai Laravel:

sudo apt install php-fpm php-cli php-mysql php-mbstring php-xml php-curl php-zip php-bcmath php-gd php-intl php-tokenizer php-common unzip curl git -y

Cek versi:

php -v
nginx -v
mysql --version

6. Install Composer
-------------------
Jalankan:

cd ~
curl -sS https://getcomposer.org/installer -o composer-setup.php
sudo php composer-setup.php --install-dir=/usr/local/bin --filename=composer
composer -V

Composer dipakai untuk install package PHP dari project Laravel.

7. Siapkan folder project
-------------------------
Buat folder aplikasi:

sudo mkdir -p /var/www/laravel-app
sudo chown -R htjogit:www-data /var/www/laravel-app
sudo chmod -R 775 /var/www/laravel-app

Masuk ke folder project:

cd /var/www/laravel-app

Catatan:
- ganti `htjogit` sesuai user server yang dipakai
- `www-data` adalah group yang umum dipakai Nginx

8. Konfigurasi Nginx
--------------------
Buat file site config:

sudo nano /etc/nginx/sites-available/laravel-app

Isi contoh:

server {
    listen 80;
    server_name 192.168.1.8;

    root /var/www/laravel-app/public;
    index index.php index.html;

    add_header X-Frame-Options "SAMEORIGIN";
    add_header X-Content-Type-Options "nosniff";

    charset utf-8;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location = /favicon.ico { access_log off; log_not_found off; }
    location = /robots.txt  { access_log off; log_not_found off; }

    error_page 404 /index.php;

    location ~ \.php$ {
        include snippets/fastcgi-php.conf;
        fastcgi_pass unix:/run/php/php-fpm.sock;
    }

    location ~ /\.(?!well-known).* {
        deny all;
    }
}

Aktifkan site:

sudo ln -s /etc/nginx/sites-available/laravel-app /etc/nginx/sites-enabled/
sudo rm -f /etc/nginx/sites-enabled/default

Test lalu restart:

sudo nginx -t
sudo systemctl restart nginx

Catatan:
- jika project belum di-clone, akses browser masih bisa error dan itu normal
- fokus langkah ini adalah memastikan konfigurasi Nginx sudah aktif

9. Cek socket PHP-FPM
---------------------
Nama socket PHP-FPM bisa berbeda tergantung versi PHP.

Cek dengan:

ls /run/php/

Contoh hasil:

php8.3-fpm.sock

Jika hasilnya spesifik seperti di atas, ubah config Nginx:

fastcgi_pass unix:/run/php/php8.3-fpm.sock;

Lalu test ulang:

sudo nginx -t
sudo systemctl restart nginx

10. Clone project dari GitHub
-----------------------------
Untuk repository public:

git clone https://github.com/USERNAME/REPO.git .

Untuk repository private dengan token:

git clone https://USERNAME:TOKEN_GITHUB@github.com/USERNAME/REPO.git .

Setelah clone, cek:

git status

Target yang aman adalah working tree bersih tanpa perubahan lokal.

11. Install dependency project
------------------------------
Install package PHP:

composer install --no-dev --optimize-autoloader

Jika project memakai Vite atau asset frontend:

sudo apt install nodejs npm -y
npm install
npm run build

Tujuan:
- `composer install` memasang package production
- `npm run build` membuat asset frontend versi production

12. Setup file .env
-------------------
Buat file environment:

cp .env.example .env
nano .env

Contoh nilai penting:

APP_NAME="Laravel App"
APP_ENV=production
APP_DEBUG=false
APP_URL=http://192.168.1.8

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=laravel_db
DB_USERNAME=laravel_user
DB_PASSWORD=password_kuat

Setelah itu generate app key:

php artisan key:generate

13. Jalankan migrasi database
-----------------------------
Jalankan:

php artisan migrate

Jika ada seeder:

php artisan db:seed

Untuk deployment production berikutnya, lebih aman memakai:

php artisan migrate --force

14. Atur permission Laravel
---------------------------
Set owner dan permission:

sudo chown -R htjogit:www-data /var/www/laravel-app
sudo find /var/www/laravel-app -type f -exec chmod 664 {} \;
sudo find /var/www/laravel-app -type d -exec chmod 775 {} \;
sudo chmod -R 775 storage bootstrap/cache

Folder yang paling penting agar writable:
- `storage`
- `bootstrap/cache`

15. Test aplikasi
-----------------
Buka di browser:

http://192.168.1.8

Jika semua benar, Laravel seharusnya sudah tampil.

Jika belum tampil, cek:
- `sudo nginx -t`
- `systemctl status nginx`
- `systemctl status php8.x-fpm`
- isi `.env`
- permission folder `storage` dan `bootstrap/cache`

16. Workflow update dari GitHub
-------------------------------
Setelah project pertama kali berjalan, update berikutnya cukup dilakukan dari server.

Contoh update lengkap jika ada perubahan backend dan frontend:

cd /var/www/laravel-app
git pull origin main
composer install --no-dev --optimize-autoloader
npm install
npm run build
php artisan migrate --force
php artisan optimize:clear
php artisan config:cache
php artisan route:cache
php artisan view:cache
sudo systemctl reload nginx

Contoh update ringan jika hanya perubahan PHP dan tidak ada perubahan frontend:

cd /var/www/laravel-app
git pull origin main
composer install --no-dev --optimize-autoloader
php artisan migrate --force
php artisan optimize:clear
php artisan optimize
sudo systemctl reload nginx

17. Script deploy satu perintah
-------------------------------
Agar deployment lebih konsisten, buat script:

nano /var/www/laravel-app/deploy.sh

Isi script:

#!/bin/bash
set -e

cd /var/www/laravel-app

echo "Pulling latest code..."
git pull origin main

echo "Installing composer dependencies..."
composer install --no-dev --optimize-autoloader

echo "Building frontend..."
npm install
npm run build

echo "Running migrations..."
php artisan migrate --force

echo "Clearing and caching Laravel..."
php artisan optimize:clear
php artisan config:cache
php artisan route:cache
php artisan view:cache

echo "Fixing permissions..."
sudo chown -R htjogit:www-data /var/www/laravel-app
sudo chmod -R 775 storage bootstrap/cache

echo "Reloading services..."
sudo systemctl reload nginx

echo "Deploy complete."

Aktifkan script:

chmod +x /var/www/laravel-app/deploy.sh

Jalankan deploy:

cd /var/www/laravel-app
./deploy.sh

18. Checklist aman sebelum update
---------------------------------
Sebelum menjalankan `git pull`, cek:

git status

Kalau hasilnya bersih, misalnya:

working tree clean

Maka update lebih aman untuk dilanjutkan.

Hindari:
- edit file source code langsung di server
- menyimpan perubahan lokal di server tanpa commit
- menjalankan migrasi production tanpa review

19. Best practice
-----------------
- Jangan edit source code di server, kecuali `.env` jika memang perlu.
- Semua perubahan aplikasi sebaiknya lewat GitHub.
- Gunakan branch deploy yang jelas, misalnya `main`.
- Simpan backup database sebelum perubahan besar.
- Jalankan `php artisan migrate --force` hanya setelah yakin migrasi aman.
- Jika pakai domain, arahkan `server_name` ke domain final lalu pasang SSL.

20. Ringkasan urutan deploy yang disarankan
-------------------------------------------
Urutan paling rapi:
1. Update server.
2. Install Nginx dan MySQL.
3. Secure MySQL.
4. Buat database dan user.
5. Install PHP, Git, Curl, Unzip, dan Composer.
6. Buat folder project.
7. Setup Nginx.
8. Cek socket PHP-FPM.
9. Clone project dari GitHub.
10. Install dependency Composer dan Node jika perlu.
11. Setup `.env`.
12. Generate key Laravel.
13. Jalankan migrasi.
14. Atur permission.
15. Test aplikasi.
16. Gunakan workflow update atau `deploy.sh` untuk deployment berikutnya.

21. Perintah inti paling sering dipakai
---------------------------------------
Update aplikasi:

cd /var/www/laravel-app
git pull origin main
composer install --no-dev --optimize-autoloader
php artisan migrate --force
php artisan optimize:clear
php artisan optimize
sudo systemctl reload nginx

Jika ada asset frontend:

npm install
npm run build
